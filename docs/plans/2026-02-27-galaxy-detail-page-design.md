# Galaxy Detail Page — Design

## Overview

A full-screen detail page at `/g/:name` showing a procedural 3D visualization of a galaxy alongside an info card overlay. Clicking a galaxy on the home sky view navigates here.

## Route & Navigation

- **Route:** `/g/:name` where name is URL-encoded galaxy name (e.g. `UGC 12345` → `/g/UGC%2012345`)
- **Lookup:** Decode slug, query by exact name match via new `getGalaxyByName()` in `useGalaxyData`
- **Entry:** Click handler on home page galaxies (extends existing hover/tooltip)
- **Exit:** Back button returns to `/`

## Architecture

### Renderer: Three.js (port from gcanvas)

Port the gcanvas `galaxy-playground` generator and renderer into TypeScript Three.js components. No gcanvas dependency.

### Visual Layers

1. **GalaxyParticles** — THREE.Points with custom shader. Stars generated by ported generator, differential Keplerian rotation each frame.
2. **GalaxyNebula** — Fullscreen quad with fragment shader porting the WebGL nebula renderer. Density map from star positions, additive blending.
3. **GalaxyBlackHole** — Small quad at galaxy center with accretion disk shader. Tilts with camera.
4. **GalaxyHaze** — Radial gradient sprite for soft galactic background glow.

### Scene Orchestrator

`GalaxyScene.ts` owns the Three.js scene, orbit camera (drag-to-tilt, scroll-to-zoom), and animation loop. Composes all visual layers.

## Data-to-Params Mapping Layer

The critical bridge between real catalog data and the procedural generator.

### Per-Type Parameter Sets (discriminated union)

Each galaxy type produces only the parameters its generator function expects:

| Type | Unique params | Shared params |
|---|---|---|
| Elliptical | `ellipticity`, `axisRatio` | `starCount`, `galaxyRadius` |
| Lenticular | `bulgeRadius`, `bulgeFraction`, `diskThickness` | `starCount`, `galaxyRadius` |
| Spiral | `numArms`, `armWidth`, `spiralTightness`, `spiralStart`, `bulgeRadius`, `fieldStarFraction` | `starCount`, `galaxyRadius` |
| Barred | `barLength`, `barWidth` + all spiral params | `starCount`, `galaxyRadius` |
| Irregular | `irregularity`, `clumpCount` | `starCount`, `galaxyRadius` |

### From Morphology String

The Hubble sequence letter (a/ab/b/bc/c/cd/d/m) maps to a T-type (1-9) that drives spiral/barred params:

- `spiralTightness`: 0.08 (Sa, tight) → 0.50 (Sm, open)
- `armWidth`: 15 (Sa) → 120 (Sm)
- `bulgeRadius`: 140 (Sa) → 0 (Sm)
- `fieldStarFraction`: 0.05 (Sa) → 0.40 (Sm)

Elliptical E-number: `E0`–`E7` → `ellipticity = N/10`, `axisRatio = 1 - N/10`

Bar strength: `SB` → strong bar (barLength ~ 0.3-0.6 × radius), `SAB` → weak bar (0.15-0.3 × radius)

Ring notation: `(r)` → high spiralStart (50-80), `(s)` → low spiralStart (10-30)

### From Other Data Fields

| Data field | Maps to | How |
|---|---|---|
| `axial_ratio` | `axisRatio` (ellipticals) | Direct mapping |
| `diameter_arcsec` + `velocity` | `galaxyRadius`, `starCount` | Physical size via Hubble law |
| `b_mag` + distance | `starCount` | Absolute magnitude → relative luminosity → particle count |
| `activity_class` | Black hole intensity | Sy1/Sy2 → brighter accretion disk |
| `position_angle` | Initial rotation | Starting angle for rendered galaxy |

### Randomized (seeded by galaxy ID)

| Param | Strategy |
|---|---|
| `numArms` | 2 for barred (80%), 2 for Sa/Sb (60%), 2-5 for Sc/Sd |
| `spiralStart` | Random 20-60 unless ring notation present |
| `clumpCount` | Random 3-8 for irregulars |
| `irregularity` | Random 0.3-0.7 for irregulars |
| `diskThickness` | Random within sub-type range for lenticulars |
| `unknown` type | Falls back to spiral with mid-range defaults |

### Implementation

- `parseMorphology(morph: string)` — regex parser → `{ type, hubbleStage, eNumber, barStrength, ringType }`
- `galaxyToGeneratorParams(galaxy: Galaxy)` — catalog data → type-specific generator params
- Uses `galaxy.id` as seed for deterministic randomization

## Info Overlay

**GalaxyInfoCard.vue** — Compact semi-transparent card (top-left):
- Galaxy name (large)
- Catalog + morphology class
- Redshift / velocity
- Key magnitudes (B, R if available)
- `pointer-events: none` except back button
